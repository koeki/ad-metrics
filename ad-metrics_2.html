Device Fingerprint (dfp) use
enable | disable i

This parameter controls wether you want to have user device fingerprinting calculated.

dfp=0 disable fingerprint event
default: 1
event config (ev) use
i| l| v| c| h| u| f| i
default: lfvc

The ev parameter specifies which event types you want to track. Depending on the privacy and data protection requirements
you may not want to track engagement device finger printing events. The value is a string with the follwing characters:

  • i: Impression (download of the metrics.js script)
  • l: Load (load and execution of metrics.js with info browser features)
  • v:Visibile (ad was visible for a period)
  • c: Click (ad was clicked)
  • h: Hover (cursor moved over the ad)
  • u: Unload (the ad or page is removed)
  • f: Fingerprint (finger print user device)

So

ev=lvc

will track the load, visble and click events. This is also the default setting.
ad size (sz) use
i

The sz parameter specifies the ad's dimensions.

sz=300x200
sz=0x0
default: 0x0 (autodetect when loaded)

The autodection of the ad's size will not work if the ad is embedded directly into the page (not served in an iframe).
It is recommended that you always set this to the correct value.
Maximum Visibility Events (mve) use
i

This parameter controls how many vis events you want to capture per impression.
If you want to report on dwell or capture visibility duration you need to omit this parameter.

mve=1 only one visibilty event will be sent
default: unlimited
Maximum Hover Events (mhe) use
i

This parameter controls how many hover events you want to capture per impression.
If you want to report on dwell or engagament time you need to omit this parameter.

mhe=1 only events for one hover interaction (in, move, out) will be sent
default: unlimited
content node (cn) use
i

The metrics javascript code needs to determine the extends and visibility bounds of the ad tag or creative to measure correctly.
Depending on how you integrate metrics - specific methods to detect the visibility need to be used.

If you integrate metrics.js in the same iframe as the actual creative or ad. Then no specific value needs to be set.

If the metrics js code is side loaded (that is the case if it is being used as a 3rd party tracking pixel) that sits outside of
the ads iframe or no iframe is used you can either specifcy the ad's dom node by passing its id directly or use one of the other methods losted below:

You must not use 0 if you cannot guarantee that the metrics code ad will be served from within the ads iframe.

cn=0 # document body (only in iframe)
cn=1 # first div element preceding the script element
cn=5 # vendor specific lookup <script src="../metrics.js"id="abcd_efgh"
     # content node will be <object id="efgh">
cn=elementId
default: 0
